## Overview

Hypedrive Brand is the dashboard brands use to run influencer marketing campaigns end-to-end:

```
Create Campaign  →  Influencers Enroll  →  Brand Reviews  →  Approve & Pay
```

Everything in one place — campaign builder, enrollment queue, wallet, team, analytics. Built with full SSR, real-time updates, multi-org support, and role-based access.

## Getting Started

### Prerequisites

- **Node.js** 20+
- **pnpm** 10+
- **Encore CLI** — [install guide](https://encore.dev/docs/install) (for the backend)

### Setup

```bash
# 1. Clone
git clone https://github.com/your-org/hypedrive-brand
cd hypedrive-brand

# 2. Install dependencies
pnpm install

# 3. Configure environment
cp .env.example .env.local
```

Edit `.env.local`:

```env
VITE_API_URL=http://localhost:4000   # Your Encore backend URL
```

### Run

```bash
pnpm dev
```

Open **http://localhost:3000** ✓

## Architecture

### SSR Execution Model

TanStack Start runs code in three distinct environments:

```
┌──────────────────┬────────────────────────────────────────┐
│ Server only      │ createServerFn(), server route handlers │
├──────────────────┼────────────────────────────────────────┤
│ Isomorphic       │ Route loaders, beforeLoad              │  ← SSR + client nav
├──────────────────┼────────────────────────────────────────┤
│ Client only      │ useEffect, localStorage, window.*      │
└──────────────────┴────────────────────────────────────────┘
```

> **Critical:** Route `loader`s are isomorphic — they run server-side on first load and **client-side** on navigation. Never put secrets or DB calls in a loader directly. Wrap them in `createServerFn()`.

```ts
// ❌ Wrong — env var exposed to client bundle
loader: () => fetch(`/api?key=${process.env.SECRET}`)

// ✅ Correct — server-only via createServerFn
const getData = createServerFn().handler(async () => {
  return fetch(`/api?key=${process.env.SECRET}`) // stays server-side
})
loader: () => getData()
```

### Authentication Flow

```
Login → JWT token → localStorage + hd_auth cookie (7d TTL)
                                         ↓
                          Server reads cookie for SSR auth checks
                                         ↓
                          401 anywhere → clear cookie + cache + store
```

2FA is handled via WebAuthn (`@simplewebauthn/browser`) — passkeys & hardware keys.

### Multi-Tenancy

Every app route lives under `/:orgSlug/`. The `_app/$orgSlug.tsx` layout:

1. Validates org belongs to the current user
2. Checks approval state → redirects to `/pending-approval` or `/rejected` if needed
3. Sets org context in router tree + Zustand store

### RBAC

```tsx
// Declarative
<Can resource="campaign" action="update">
  <EditButton />
</Can>

// Imperative
const canInvite = useCan("team", "invite");
```

Permissions resolved server-side and synced to `permissions-store` on mount.
